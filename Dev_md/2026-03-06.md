## DreamIT Biz Docs 프로젝트 초기 구축

### 개요
분야별 학습 자료를 정리하고 브라우저에서 PDF를 바로 열람할 수 있는 문서 라이브러리 사이트를 구축했다.

### 기술 스택
- **프론트엔드**: React 19 + React Router 7
- **빌드 도구**: Vite 6
- **배포**: GitHub Pages (GitHub Actions 자동 배포)
- **도메인**: docs.dreamitbiz.com

### 프로젝트 구조
```

```

## 관리자 CRUD + 회원 관리 시스템 추가

### 개요
정적 데이터(documents.js)만 사용하던 사이트에 Supabase 기반 관리자 CRUD 게시판과 회원 관리 기능을 추가했다. Supabase 미설정 시 기존 정적 데이터로 정상 동작하는 graceful fallback 구조.

### 추가 기술 스택
- **백엔드/DB**: Supabase (PostgreSQL + Auth + Storage)
- **패키지 추가**: `@supabase/supabase-js`

### 구현 단계

#### Phase 1: Foundation (기반 구축)
- **`src/utils/supabase.js`** - Supabase 클라이언트 (env 없으면 null 반환)
- **`src/config/admin.js`** - 관리자 이메일 화이트리스트 (`aebon@dreamitbiz.com`)
- **`src/services/documents.js`** - 데이터 추상화 레이어
  - Supabase 연결 시 DB에서 조회, 미연결 시 정적 데이터 fallback
  - CRUD 함수: `fetchDocuments`, `createDocument`, `updateDocument`, `deleteDocument`, `deleteDocuments`, `seedDocuments`
  - 검색/집계: `searchDocumentsAsync`, `fetchCategoryCounts`
- **`.env.example`** - 환경변수 템플릿

#### Phase 2: Auth (인증 시스템)
- **`src/contexts/AuthContext.jsx`** - 인증 상태 관리
  - `signInWithEmail`, `signOut`, `isAdminUser` 제공
  - Supabase Auth 세션 리스닝
- **`src/components/auth/LoginModal.jsx`** - 이메일/비밀번호 로그인 모달
- **`src/components/auth/AuthGuard.jsx`** - 관리자 전용 라우트 보호 (비관리자 → 홈 리다이렉트)
- **`src/styles/auth.css`** - 모달/폼 스타일 (다크모드 대응)

#### Phase 3: Admin Shell (관리자 셸)
- **`src/layouts/AdminLayout.jsx`** - 사이드바 + Outlet 레이아웃
  - 네비게이션: 대시보드, 문서 관리, 회원 관리
  - 사용자 정보 표시, 로그아웃 버튼
  - 모바일 반응형 드로어
- **`src/pages/admin/AdminDashboard.jsx`** - 대시보드
  - 통계 카드 (전체 문서, 활성 카테고리, 전체 카테고리)
  - 최근 문서 테이블
  - 카테고리별 현황 막대 차트
- **`src/styles/admin.css`** - 관리자 UI 스타일 시스템 (레이아웃, 테이블, 폼, 버튼, 배지, 툴바)

#### Phase 4: Document CRUD (문서 관리)
- **`src/utils/storage.js`** - Supabase Storage PDF 업로드/삭제
- **`src/pages/admin/AdminDocuments.jsx`** - 문서 목록 페이지
  - 검색, 체크박스 선택, 벌크 삭제
  - 더미 데이터 시딩 버튼
  - 행별 수정/삭제
- **`src/pages/admin/AdminDocumentForm.jsx`** - 문서 생성/수정 폼
  - 슬러그, 카테고리, 제목, 설명, PDF 업로드, 페이지 수, 태그, 추가일

#### Phase 5: Member Management (회원 관리)
- **`src/services/members.js`** - 프로필 조회 (`fetchMembers`, `fetchMemberCount`)
- **`src/pages/admin/AdminMembers.jsx`** - 회원 목록 페이지
  - 아바타, 이름, 이메일, 관리자 배지, 가입일
  - 이름/이메일 검색

#### Phase 6: Public Page Migration (공개 페이지 전환)
- **Home, Category, Viewer** → `useState`/`useEffect` 비동기 데이터 로딩으로 전환
- **Navbar** → `searchDocumentsAsync` 비동기 검색 + 로그인/관리자 버튼 추가
- **CategoryCard** → `docCount` prop으로 카운트 외부 주입

### Supabase 테이블 스키마
```sql
-- documents 테이블
CREATE TABLE documents (
  id          BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  slug        TEXT UNIQUE NOT NULL,
  category    TEXT NOT NULL,
  title       TEXT NOT NULL,
  description TEXT DEFAULT '',
  file_name   TEXT DEFAULT '',
  file_url    TEXT DEFAULT '',
  file_size   TEXT DEFAULT '',
  pages       INT,
  tags        TEXT[] DEFAULT '{}',
  added_at    DATE DEFAULT CURRENT_DATE
);

-- profiles 테이블 (auth trigger로 자동 생성)
CREATE TABLE profiles (
  id          UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email       TEXT,
  full_name   TEXT,
  avatar_url  TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- auth trigger
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, email)
  VALUES (NEW.id, NEW.email);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
AFTER INSERT ON auth.users
FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```

## 소셜 로그인 (카카오, Google) 추가

### 개요
기존 이메일/비밀번호 전용 로그인 모달에 카카오, Google OAuth 소셜 로그인을 추가했다. www.dreamitbiz.com과 동일한 소셜 로그인 방식 적용.

### 변경 내용

#### AuthContext 확장
- `signInWithGoogle()` - Supabase OAuth `google` provider
- `signInWithKakao()` - Supabase OAuth `kakao` provider
- 두 함수 모두 `redirectTo: window.location.origin` 으로 로그인 후 현재 사이트로 복귀

#### LoginModal 리디자인
- **카카오 버튼** - 노란색(#FEE500) 배경, 카카오 말풍선 아이콘
- **Google 버튼** - 흰색 배경 + 구글 4색 로고
- **"또는" 구분선** - 소셜 로그인과 이메일 로그인 사이 시각적 분리
- 기존 이메일/비밀번호 폼은 하단에 유지

#### 스타일 추가 (auth.css)
- `.auth-social` - 소셜 버튼 컨테이너
- `.auth-social-btn` - 공통 소셜 버튼 스타일
- `.auth-kakao` - 카카오 전용 (노란 배경 + 검정 텍스트)
- `.auth-google` - Google 전용 (흰 배경 + 테두리, 다크모드 대응)
- `.auth-divider` - "또는" 구분선 (좌우 라인 + 중앙 텍스트)
